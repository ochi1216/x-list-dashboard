# CLAUDE.md

このファイルは、Claude Code Web によるセッション間の作業引き継ぎのための運用ルールを定義する。

## プロジェクト名

X List Dashboard

（注: このリポジトリ名は `x-list-dashboard`。過去のセッション指示で「YouTube Summary Tool」という名称が使われたことがあるが、リポジトリの実内容（Xの複数アカウント投稿を一覧・既読管理・読み上げする静的ダッシュボード）とは一致しないため不採用。今後のセッションでは本ファイルの「X List Dashboard」を正式名称として扱う。）

## セッション運用

* このプロジェクトは「1タスク＝1セッション」で進める。
* 各セッション開始時に、以下のファイルを必ず読む。
  * `CLAUDE.md`
  * `docs/PROJECT_STATUS.md`
  * `docs/SESSION_HISTORY.md`
  * `docs/NEXT_TASK.md`
* 原則として、`docs/NEXT_TASK.md` に書かれた1つのタスクだけを実施する。
* タスクの範囲を勝手に拡大しない。
* 関係のないファイルを変更しない。
* 不明点や確認できない事項を推測で補完しない。
* 確認できない事項は「未確認」と記載する。

## セッションタイトル

* セッション開始時に、推奨セッションタイトルを最初に提示する。
* タイトル形式は次のとおりとする。

  `プロジェクト名 S連番 - 今回のタスク`

* セッション番号は `docs/SESSION_HISTORY.md` の最新番号に1を加える。
* 初回は `S01` とする。
* プロジェクト名の表記（`X List Dashboard`）を途中で変更しない。
* タスク部分は簡潔に記載する。

## 作業終了時

作業完了時には、必ず以下を実施する。

1. 必要に応じて `docs/PROJECT_STATUS.md` を更新する。
2. `docs/SESSION_HISTORY.md` に今回の作業履歴を追記する。
3. `docs/NEXT_TASK.md` を次回作業用に更新する。
4. 変更したファイルを一覧表示する。
5. 実施したテストと結果を報告する。
6. 未実施または未確認の項目を明記する。
7. 次回の推奨セッションタイトルを提示する。
8. 次のセッションに貼り付ける短い開始プロンプトを作成する。

## Git運用

* 現在のブランチを確認してから作業する。
* ユーザーから指示がない限り、既存の安定版機能を削除しない。
* 作業終了前に差分を確認する。
* テスト可能な変更はテストする。
* テストできなかった場合は理由を記録する。
* 秘密情報、APIキー、パスワード、認証情報をコミットしない。
  * ただし `index.html` 内の Supabase URL / publishable key は、クライアントサイドで動作する RLS 保護前提の公開用キーとして意図的にハードコードされている（2026-07-16時点）。これを機密情報として扱う必要はないが、権限の広い secret key を誤って埋め込まないよう注意する。
